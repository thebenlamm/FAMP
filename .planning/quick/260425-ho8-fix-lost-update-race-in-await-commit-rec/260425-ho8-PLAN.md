  <behavior>
    Add a fallible sibling of `TaskDir::update` that lets callers run a `Result`-returning mutation (e.g., FSM advance) atomically inside the same read+write critical section, with three contracts:

    1. **Happy path**: closure returns `Ok(record)` → record is atomically persisted (same `write_atomic_file` path as `update`); returns `Ok(persisted_record)`.
    2. **Closure-Err path**: closure returns `Err(E)` → NO disk write occurs (file bytes unchanged); returns `Err` carrying the closure's `E` value, distinguishable from a `TaskDirError`.
    3. **Identity-stability path**: closure returns `Ok(record)` whose `task_id` differs from the input → returns `TaskDirError::TaskIdChanged` (same invariant as `update`); no write.
    4. **Read failure**: if the initial read fails (`NotFound`, `Io`, `TomlParse`), no closure is ever invoked; returns the read error.

    API shape (chosen to match the `Result`-bearing CliError-friendly contract `await_cmd` needs):

    ```rust
    /// Error type for `try_update`. Wraps either a closure-returned `E`
    /// or a `TaskDirError` from the underlying read/write/validation path.
    /// Variants kept narrow per CLAUDE.md ("phase-appropriate error enums").
    #[derive(Debug, thiserror::Error)]
    pub enum TryUpdateError<E> {
        /// Closure returned an error — atomic write was NOT performed.
        #[error("update closure failed")]
        Closure(#[source] E),

        /// Underlying store error (read, validation, write).
        #[error(transparent)]
        Store(#[from] TaskDirError),
    }

    impl TaskDir {
        /// Read → fallible-mutate → atomic write. Atomic with respect to the
        /// read step: the closure receives the on-disk record and the persisted
        /// bytes are derived from the closure's returned record (NOT a stale
        /// snapshot from a separate `read()` call).
        ///
        /// On `Err` from the closure, NO disk write occurs; the original file
        /// is byte-identical to its pre-call state.
        ///
        /// Reuses the same internal write path as [`Self::update`]; do not
        /// duplicate `write_atomic_file` / id-stability / serialization code.
        pub fn try_update<E, F>(
            &self,
            task_id: &str,
            mutate: F,
        ) -> Result<TaskRecord, TryUpdateError<E>>
        where
            F: FnOnce(TaskRecord) -> Result<TaskRecord, E>,
        {
            let current = self.read(task_id)?;            // Store err → ?
            let next = mutate(current).map_err(TryUpdateError::Closure)?;
            if next.task_id != task_id {
                return Err(TryUpdateError::Store(TaskDirError::TaskIdChanged {
                    original: task_id.to_string(),
                    next: next.task_id,
                }));
            }
            let path = self.path_for(task_id)?;
            let body = toml::to_string(&next).map_err(|source| {
                TryUpdateError::Store(TaskDirError::TomlSerialize {
                    task_id: next.task_id.clone(),
                    source,
                })
            })?;
            write_atomic_file(&path, body.as_bytes()).map_err(|source| {
                TryUpdateError::Store(TaskDirError::Io {
                    path: path.clone(),
                    source,
                })
            })?;
            Ok(next)
        }
    }
    ```

    Note: this DOES re-use `read`, `path_for`, `write_atomic_file` — same atomicity/locking semantics as `update`. The existing `update` is left untouched (backward compatible).
