<objective>
Fix the stop-hook blind spot: when `famp await` returns a channel message, the hook currently says "Call famp_inbox to read it." but famp_inbox only reads DM mailboxes. Channel messages are invisible to it.

The fix has three interlocked parts:
1. Change `write_outcome` to emit a single wrapper JSON object (not raw JSONL per envelope) so the hook can extract mailbox kind/name.
2. Populate `wait_reply.rs`'s inbox-found path with a real `MailboxName::Agent(identity)` so the wrapper always has a valid mailbox when envelopes are non-empty.
3. Update the hook's Python META extraction and Bash notification message to be channel-aware.
4. Update the `wait_reply.rs` integration test to parse the new wrapper format.

Purpose: Agents woken by the stop hook receive a notification that correctly directs them to famp_channel_log (channels) or famp_inbox (DMs) — no more confusion.
Output: 4 files changed; cargo nextest passes; just install run.
</objective>

**In `crates/famp/src/cli/await_cmd/mod.rs` — `write_outcome` function (lines 125–148):**

Replace the current implementation that prints one envelope per line with a single wrapper object:

```rust
pub(crate) fn write_outcome(outcome: &AwaitOutcome, mut out: impl Write) -> Result<(), CliError> {
    if outcome.timed_out {
        let mut value = serde_json::json!({"timeout": true});
        if let Some(diagnostic) = outcome.diagnostic.as_deref() {
            value["diagnostic"] = serde_json::json!(diagnostic);
        }
        writeln!(out, "{value}").map_err(|e| CliError::Io {
            path: std::path::PathBuf::new(),
            source: e,
        })?;
    } else {
        let mailbox_value = match &outcome.mailbox {
            Some(MailboxName::Channel(name)) => {
                serde_json::json!({"kind": "channel", "name": name})
            }
            Some(MailboxName::Agent(name)) => {
                serde_json::json!({"kind": "agent", "name": name})
            }
            None => serde_json::Value::Null,
        };
        let wrapper = serde_json::json!({
            "mailbox": mailbox_value,
            "envelopes": outcome.envelopes,
            "next_offset": outcome.next_offset,
        });
        let line = serde_json::to_string(&wrapper).map_err(|e| CliError::Io {
            path: std::path::PathBuf::new(),
            source: std::io::Error::new(std::io::ErrorKind::InvalidData, e),
        })?;
        writeln!(out, "{line}").map_err(|e| CliError::Io {
            path: std::path::PathBuf::new(),
            source: e,
        })?;
    }
    Ok(())
}
```

import json, sys
count = 0
sender = "unknown"
mailbox_kind = "agent"
mailbox_name = ""
for line in sys.argv[1].splitlines():
    line = line.strip()
    if not line:
        continue
    try:
        env = json.loads(line)
    except Exception:
        continue
    if env.get("timeout") is True:
        continue
    if isinstance(env.get("envelopes"), list):
        mb = env.get("mailbox") or {}
        if isinstance(mb, dict):
            mailbox_kind = mb.get("kind", "agent")
            mailbox_name = mb.get("name", "")
        for item in env["envelopes"]:
            if isinstance(item, dict):
                count += 1
                sender = item.get("from", item.get("sender", "unknown"))
        continue
    # Fallback: raw envelope line (backward compat)
    count += 1
    sender = env.get("from", env.get("sender", "unknown"))
    mailbox_kind = "agent"
    mailbox_name = ""
print(f"{count}|{sender}|{mailbox_kind}|{mailbox_name}")
' "$MSG" 2>/dev/null || printf '1|unknown|agent|\n')"

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| broker→CLI stdout | AwaitOk envelopes from broker serialized into wrapper JSON |
| hook←CLI stdout | Hook reads wrapper JSON, extracts mailbox kind/name for notification only |
| notification→Claude | Reason string: peer-controlled bytes must NOT appear here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-s3h-01 | Tampering | MAILBOX_NAME in notification string | mitigate | Channel name comes from the broker (not the envelope sender); validated by existing identity rules at join time. The name appears only in the famp_channel_log({channel: '...'}) call hint, not as executable code. |
| T-s3h-02 | Information Disclosure | famp-await.sh REASON string | accept | REASON contains only count, sender identity (grep-validated to `^[A-Za-z0-9@._:/-]{1,128}$`), and channel name. Peer envelope bytes are explicitly excluded per existing SECURITY comment in the hook. |
| T-s3h-03 | Spoofing | Malformed wrapper JSON from write_outcome | accept | write_outcome is Rust code in the trusted famp binary; serde_json serialization is deterministic. The hook's Python parser handles malformed JSON with a try/except that falls back to the raw-envelope branch. |
</threat_model>
