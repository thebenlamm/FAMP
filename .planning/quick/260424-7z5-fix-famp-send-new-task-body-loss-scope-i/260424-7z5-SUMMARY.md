# Phase 260424-7z5: Fix `famp_send` new_task body loss (scope.instructions) Summary

**`famp send --new-task --body <prose>` now lands the body on the wire under `body.scope.instructions`; prior behaviour silently emitted `scope:{}` and forced a second `deliver` workaround. Locked by a TDD regression test, centralised behind a named constant, and captured in the repo's first ADR (PROVISIONAL) plus a provisional conformance vector.**

## Accomplishments

- **Beta-feedback Issue 1 closed.** `famp send --new-task "<title>" --body "<prose>" --to <peer>` now writes a signed request envelope whose `body.scope.instructions` equals `<prose>` exactly; `body.natural_language_summary` still equals `<title>` (title path unchanged).

## Decisions Made

- **Key name `instructions`** over six rejected alternatives — see ADR 0001 for full rationale. The selection leans on LLM-era API-surface familiarity (Anthropic/OpenAI) and self-documenting semantics.
- **PROVISIONAL status, not normative.** The v0.5.1 spec fork explicitly does NOT gain this convention in this task. The exit ramp is the single constant + the ADR re-evaluation gate. The fixture lives under `tests/fixtures/provisional/` specifically so future Level 2 conformance loaders ignore it.
- **`RequestBody.scope` stays `serde_json::Value`-typed.** Re-typing to a `{ Instructions(String), Structured(Value) }` enum is listed in the ADR re-evaluation candidates but deferred until the convention is validated by real usage.
- **Body-less sends preserve the `scope:{}` shape.** This is a deliberate backward-compat choice — every existing test asserting on `scope == {}` stays byte-identical.
- **The TDD RED commit is preserved, not squashed.** The commit history IS the TDD evidence; squashing would destroy the "fails first on the exact assertion the fix turns green" chain that matters for cross-agent-verifiable claims about this fix.
