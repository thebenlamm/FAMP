# Quick Task 260506-c65: Wire famp-await.sh into install-claude-code

## What was done

`famp-await.sh` (the listen-mode Stop hook) was missing from the `famp install-claude-code` distribution. Users who ran the installer got no listen mode — the hook existed only on the dev machine.

**Task 1 — Asset + await_hook module** (`e2cf7fe`)
- Copied `~/.claude/hooks/famp-await.sh` → `crates/famp/assets/famp-await.sh`
- Created `crates/famp/src/cli/install/await_hook.rs` mirroring `hook_runner.rs`: `FAMP_AWAIT_SH` const via `include_str!`, `install_shim`/`remove_shim`, 6 unit tests
- Declared `pub mod await_hook` in `install/mod.rs`

**Task 2 — Install orchestrator** (`89ecea5`)
- Extended `install/claude_code.rs` from 4 to 5 artifacts
- `await_shim_path = ~/.claude/hooks/famp-await.sh` installed at [5/5]
- Generalized `remove_famp_hook_from_stop_entry` to accept `&[String]` slice — deduplicates both famp-owned paths on re-install
- `build_stop_array` now appends two Stop entries: hook-runner.sh (timeout 30) and famp-await.sh (timeout 86400)
- Updated + added unit tests: 5-artifact assertion, double-install idempotency (exactly 2 entries), stale-entry replacement count corrected to 4

**Task 3 — Uninstall orchestrator** (`489e362`)
- Extended `uninstall/claude_code.rs` from 4 to 5 steps
- Removes await shim at [4/5], renumbered hook-runner to [3/5], settings removal to [5/5]
- Same `&[String]` generalization in `surgical_remove_stop_entry` — removes both famp Stop entries
- All 5 uninstall unit tests pass, including `uninstall_after_install_returns_to_clean_state` asserting await shim absent

**Task 4 — Integration test + full sweep** (`54fcb47`)
- Fixed `crates/famp/tests/install_claude_code.rs::install_claude_code_writes_all_artifacts` (was asserting 1 Stop entry; now asserts 2 with correct timeouts and paths, plus await shim presence at mode 0755)
- 214/214 tests pass, 1 skipped (pre-existing)

## Key design decision

`remove_famp_hook_from_stop_entry` now takes `&[String]` in both install and uninstall — the load-bearing fix identified by Zed review. Without this, uninstall would orphan the await Stop entry in settings.json.
