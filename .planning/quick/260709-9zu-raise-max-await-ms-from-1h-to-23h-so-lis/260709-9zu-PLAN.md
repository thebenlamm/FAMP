<objective>
Raise the broker's await ceiling `MAX_AWAIT_MS` from 1 hour to 23 hours so
listen windows stop going deaf ~1 hour after their last turn.

Purpose: The FAMP Stop hook runs `famp await --as <id> --timeout 23h`, but
awaiting.rs:70 silently clamped every await to 1 hour. On timeout the hook
exits 0, and Claude Code Stop hooks do not re-fire until the next turn — which
an idle listen window never has. Every listen window went deaf 1 hour after
its last turn (54 silent "clean stop" events observed in the hook log).

Output: One production-value change (the constant + its comments) plus one
unit test proving an await requesting more than 1 hour is no longer clamped.
</objective>

Key facts (verified by two independent reviewers — do NOT re-derive or "improve" on these):
- awaiting.rs:45-64 — a fresh await drains any backlog immediately BEFORE parking,
  so raising the cap opens no gap-hole.
- handle.rs:888,900 — disconnect prunes the parked await on socket close, NOT at
  deadline. A 23h cap leaks zero stale registrations for dead windows; the 1h cap
  was never doing cleanup, disconnect is.
- The tick sweep (handle.rs:929-940) is a fixed 1s `tokio::time::interval`, not
  per-await timers, so a longer deadline costs nothing.
- `ParkedAwait { filter, deadline }` struct size is deadline-value independent.
- Rust `Instant` on Darwin freezes during system suspend, so a parked await burns
  zero budget while the laptop sleeps; it can never expire early.
