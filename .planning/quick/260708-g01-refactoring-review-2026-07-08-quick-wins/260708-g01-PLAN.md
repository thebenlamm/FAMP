## Task 3 — Warn (do NOT cap) on an oversized register/join drain

**files:** `crates/famp-bus/src/broker/handle.rs`

**action:**
`decode_lines` (`handle.rs:983`) runs with `cap: None` on the register
(`:317`) and join (`:713`) paths. Its output becomes `RegisterOk.drained` /
`JoinOk.drained`, encoded into a single reply frame checked against
`MAX_FRAME_BYTES = 16 MiB` (`famp-bus/src/codec.rs:6,26`). A mailbox that crosses
16 MiB therefore makes **registration itself fail**, with no prior signal.

**Add a warn, not a cap.** Capping would truncate the drain and silently change
register/join semantics; that is a real design change and belongs with 999.11's
retention work, not a quick win.

Introduce `const DRAIN_WARN_BYTES: u64 = 8 * 1024 * 1024;` in `handle.rs`
(half the frame limit) and, in `decode_lines`, sum the drained record byte span
and emit **one** `tracing::warn!` when it exceeds the threshold:

```rust
tracing::warn!(
    mailbox = %mailbox,
    drained_bytes = total,
    limit = famp_bus_codec_max_frame_bytes,  // use the real MAX_FRAME_BYTES const
    "mailbox drain approaching the 16 MiB reply-frame limit; registration will \
     fail once it is exceeded (no retention/compaction exists yet — see 999.11)"
);
```

Compute `total` from the already-carried per-record offsets
(`records.last().end - start_offset`) — do **not** re-derive framing math; that is
exactly what §3.2 removed. Guard against `records.is_empty()`. No `unwrap`/`expect`.

Keep `cap: None`, keep the missing self-authored skip, keep the existing
per-record WARN on undecodable lines. Behavior is otherwise unchanged.

Add one unit test in `handle/tests.rs` asserting the warn threshold logic is a
pure function of `(start_offset, records)` — extract the size computation into a
small `fn drained_span(start_offset: u64, records: &[DrainedRecord]) -> u64` if
that makes it testable without a tracing subscriber. Do not add a tracing-capture
dependency to `famp-bus` (it must stay tokio-free and dep-free per BUS-01).

**verify:**
```
grep -n 'DRAIN_WARN_BYTES' crates/famp-bus/src/broker/handle.rs
cargo test -p famp-bus
just check-no-tokio-in-bus
git diff --name-only -- crates/famp-bus/tests/prop04_drain_completeness.rs crates/famp-bus/tests/tdd02_drain_cursor_order.rs   # MUST be empty
```

**done:** oversized drain emits exactly one WARN; `cap: None` preserved; famp-bus
gains no deps; pinned property tests untouched and green.

**commit:** `feat(260708-g01): warn before a register drain hits the 16 MiB frame cliff`

## Task 4 — Deduplicate canonical-holder resolution in `BrokerState::view()`

**files:** `crates/famp-bus/src/broker/state.rs`, `crates/famp-bus/src/broker/identity.rs`

**action:**
`state.rs:~150-165` re-implements canonical-holder-by-`bind_as` lookup inline as a
`find_map` over `self.clients`. `identity::canonical_holder_id` already does this.
They are semantically identical **today** — that is the drift risk.

The blocker is signature shape: the `identity::*` helpers take `&Broker<E>` while
`view()` is a `&self` method on `BrokerState`. Three of the four helpers only ever
touch `broker.state`. **Narrow the signatures to `&BrokerState`** and update the
call sites in `handle.rs` / `awaiting.rs` to pass `&broker.state`. Then have
`view()` call `identity::canonical_holder_id(self, &name)` and delete the inline
`find_map`.

If any helper genuinely needs `&Broker<E>` (i.e. touches `broker.env`), leave that
one alone and narrow only the ones that don't. Report which in the SUMMARY.

Zero behavior change. If narrowing forces a non-trivial refactor of `handle.rs` or
`awaiting.rs` beyond mechanical `broker` → `&broker.state` at call sites, STOP and
report — that is a bigger change than a quick win and needs its own task.

**verify:**
```
grep -n 'find_map' crates/famp-bus/src/broker/state.rs   # must be empty
cargo test -p famp-bus
cargo test -p famp --lib
cargo clippy --workspace --all-targets -- -D warnings
```

**done:** `view()` calls `identity::canonical_holder_id`; no inline `find_map` in
`state.rs`; all tests green; no behavior change.

**commit:** `refactor(260708-g01): BrokerState::view reuses identity::canonical_holder_id`
