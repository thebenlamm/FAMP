**Modes / flags (parsed BEFORE doing any work):**
- `--help` (or `-h`) — print usage, exit 0
- `--dry-run` (alias `--check`) — print what would happen, take no destructive action
- `--force` — skip the in-flight task safety check
- `--no-rebuild` — skip the `cargo install` step but still cycle daemons
- (no args) — interactive mode: print plan, prompt before destructive steps

**Execution flow (in order):**

1. **Repo-clean guard** (skip if `--no-rebuild`):
   - Run `git status --porcelain crates/famp/`. If any output → `die` with helpful message ("uncommitted changes in crates/famp/; commit or stash first, OR pass --no-rebuild").
   - This refuses to force-rebuild from dirty state (footgun protection).

2. **In-flight task safety guard** (skip if `--force`):
   - Enumerate `$STATE_ROOT/agents/*/tasks/*.toml` (use `find` or shell glob with `nullglob` so empty doesn't error).
   - For each TOML: extract the `state = "..."` line via `awk -F'"' '/^state[[:space:]]*=/ { print $2 }'` (mirror famp-local's `port_of()` style).
   - If state is NOT one of `{COMPLETED, FAILED, CANCELLED}` → record as in-flight. Capture the agent name (parent dir of `tasks/`) and task_id for the operator-facing message.
   - If any in-flight tasks exist: print the list ("In-flight tasks would be interrupted: …"), then `die` with hint to pass `--force` if operator owns the consequences.

3. **Discover daemons:**
   - Enumerate `$STATE_ROOT/agents/*/daemon.pid` (use `nullglob`; if zero matches, log "no listeners found" and exit 0 — successful no-op).
   - For each PID file:
     - Read PID; check `is_alive`. If dead → log "stale PID file for <agent> (pid=$pid)"; remove the stale file; do NOT add to restart list.
     - If alive → add `(name, pid, home, log)` tuple to the restart list.

4. **Print plan + interactive prompt** (skip the prompt under `--dry-run`; in `--dry-run` mode, print the plan and exit 0 here):
   - Show: agents to cycle, rebuild yes/no, log path for each.
   - If interactive (no `--force`, no `--dry-run`, stdin is a tty): prompt "Proceed? [y/N]"; abort if not `y|Y|yes`.

5. **Rebuild step** (skip if `--no-rebuild`):
   - Run `cargo install --path crates/famp` from the repo root. Stream output to stderr. If non-zero → `die` ("rebuild failed; daemons left untouched").
   - Repo root must be discovered via `git rev-parse --show-toplevel` (so the script works from any cwd).

6. **Stop daemons (SIGTERM + poll + SIGKILL fallback):**
   - For each entry: send `kill -TERM "$pid"`. Poll `is_alive` every 100ms up to a configurable timeout (default 8 seconds — derive `MAX_TRIES=80` if poll interval is 100ms).
   - If still alive after timeout → log warning, send `kill -KILL "$pid"`, poll once more.
   - On clean exit, remove the pid file.
   - Record per-agent stop result (clean / killed / failed).

7. **Restart daemons:**
   - For each entry, capture restart timestamp via `date +%s` (the boundary used to decide "fresh" beacon line in the log).
   - Launch with: `FAMP_HOME="$home" nohup "$FAMP_BIN" listen >>"$logf" 2>&1 &` — note the **APPEND** redirect (`>>`), unlike famp-local which truncates. The append preserves restart history; the task brief mandates this.
   - Capture new PID via `$!`, write to `$pid_for(name)`.

8. **Verify restart success:**
   - For each restarted daemon, poll the log file (per-daemon 10s timeout, 100ms interval).
   - "Success" = a NEW line containing `listening on https://` appears in `$logf` AFTER the restart timestamp recorded in step 7. Use `awk` or `tail -n +<line-count-before-restart>` to scope to fresh lines (not lines from a prior boot).
   - Record per-agent restart result (success / timeout / process-died).

9. **Final summary:**
   - Print a table: agent | stop-result | restart-result | pid | log-path.
   - Print "All N agents cycled cleanly" or list any failures with: PID file, log file, suggested commands (`tail -50 $logf`, `kill -0 $pid`).
   - Exit 0 if all succeeded; exit 1 if any failed.

**Non-negotiable safety guards** (re-state in script comments at top):
- DO NOT run if `crates/famp/` is dirty (unless `--no-rebuild` skips the rebuild entirely)
- DO NOT proceed if non-terminal tasks exist (unless `--force`)
- DO NOT truncate `daemon.log` — append only (`>>`)
- DO NOT hard-code agent names — discover from `$STATE_ROOT/agents/*/`
- DO NOT modify `scripts/famp-local`
- DO NOT modify daemon code or `famp listen`

**Shellcheck compliance:**
- Run `shellcheck scripts/redeploy-listeners.sh` after writing. Fix all warnings, OR document each suppression with `# shellcheck disable=SC<n>  # reason: <one-line justification>` at the suppression site.
- Acceptable suppressions: `SC2086` for intentional word-splitting on a space-separated list (mirror famp-local line 587); document the reason.

**Idempotency:**
- Running the script twice in a row should rebuild + restart twice (operator intent), but should NOT corrupt state, leave orphan PIDs, or double-write tracking files. The pid file is overwritten on each restart; the log file is appended; tasks/ on disk is untouched. This is idempotent enough.
