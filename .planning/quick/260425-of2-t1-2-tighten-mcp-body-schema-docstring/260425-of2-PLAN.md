# Quick Task 260425-of2: T1.2 — Tighten MCP `body` schema docstring

## Background

After T1.1 (quick-260424-7z5) lifted `scope.instructions` so `body` is the canonical
carrier for `new_task` content, the MCP tool schema's `body` field still describes
itself as `"Message content"` — too generic. An LLM agent reading the schema cannot
tell that:

- on `new_task`, `body` is the only place real task content lands (title is summary only),
- on `deliver`/`terminal`, `body` is the reply text.

Source of truth: `~/.claude/plans/ok-now-analyze-and-toasty-waffle.md` (T1.2 / Fix B3).

## Truths

- Current line: `crates/famp/src/cli/mcp/server.rs:47` — `"description": "Message content"`
- The MCP `tools/list` handler is exercised end-to-end by `crates/famp/tests/mcp_stdio_tool_calls.rs`
- `insta` is not currently a dev-dep of `famp`; introducing it for one test is overkill —
  a string-contains assertion in the existing subprocess test gives equivalent
  protection against regression with zero new dependencies.

## Tasks

### T1: Tighten the `body` field description (server.rs:47)

**File:** `crates/famp/src/cli/mcp/server.rs`

**Action:** Replace the current generic description with one that tells an LLM agent
the truth about the field's role across modes:

```
Task body content (the actual instructions). REQUIRED for new_task to carry content;
the title field is only a short summary. For deliver/terminal modes, this is the
reply text.
```

**Verify:** `grep -n "Task body content (the actual instructions)" crates/famp/src/cli/mcp/server.rs`
returns line 47.

**Done when:** description text replaced; `cargo build -p famp` succeeds.

### T2: Add tools/list body-description assertion to mcp_stdio_tool_calls.rs

**File:** `crates/famp/tests/mcp_stdio_tool_calls.rs`

**Action:** In the existing `tools/list` test path (around line 191), after parsing the
response, add an assertion that the `body` schema property's `description` contains the
new tightened text. Specifically, assert that the description contains the substring
`"REQUIRED for new_task"` so accidental reverts to the generic `"Message content"` fail
loudly without locking the wording verbatim.

**Verify:** `cargo test -p famp --test mcp_stdio_tool_calls` passes.

**Done when:** new assertion present; full workspace test suite green
(`cargo test --workspace`).
