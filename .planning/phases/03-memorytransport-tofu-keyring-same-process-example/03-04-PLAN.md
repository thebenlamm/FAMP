**Step A — CONF-07 fixture construction.**

Create `crates/famp/tests/fixtures/conf-07-canonical-divergence.json`. The simplest construction: start from a known-good canonical signed envelope (copy from `crates/famp-envelope/tests/vector_zero.rs` or similar), then reorder the top-level keys so they are NOT in alphabetical order. The signature remains valid in ed25519-dalek terms (it was computed over the canonical form), but the wire bytes are not in canonical form. When `process_one_message` re-canonicalizes the wire bytes and compares, `re_canonical != msg.bytes`, so `CanonicalDivergence` fires.

```rust

    //    serde_json::to_vec_pretty (pretty adds whitespace — also causes

}
```
