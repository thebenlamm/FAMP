<objective>
Stand up the `famp-gateway` crate skeleton and the core Design A mechanism: back each remote principal with one plain-`Register` UDS connection carrying the gateway's own `std::process::id()`. This is the spine every later v1.0 phase depends on.

Purpose: Resolve the same-host `kill(pid,0)` liveness fork structurally (LIVE-01/LIVE-02) and provide the per-principal demux table that makes cross-talk impossible (GW-04), with zero `famp-bus` change.
Output: A compiling `famp-gateway` lib + minimal `[[bin]]`, plus a no-spawn `BusClient` constructor the gateway uses to fail loud instead of auto-spawning a broker.
</objective>
