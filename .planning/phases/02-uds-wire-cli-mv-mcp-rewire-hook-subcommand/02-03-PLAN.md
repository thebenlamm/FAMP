    Run function — full reconnect loop with bounded exponential backoff (cap 30s per RESEARCH §2 item 8):
    ```rust
    pub async fn run(args: RegisterArgs) -> Result<(), CliError> {
        let sock = crate::bus_client::resolve_sock_path();
        let mut delay = std::time::Duration::from_secs(1);
        let cap = std::time::Duration::from_secs(30);
        loop {
            // Always attempt to spawn-if-absent before connect
            let _ = crate::bus_client::spawn::spawn_broker_if_absent(&sock);
            match crate::bus_client::BusClient::connect(&sock).await {
                Ok(mut client) => {
                    let pid = std::process::id();
                    let reply = client.send_recv(BusMessage::Register {
                        name: args.name.clone(), pid
                    }).await.map_err(map_bus_client_err)?;
                    match reply {
                        BusReply::RegisterOk { drained, joined } => {
                            eprintln!(
                                "registered as {} (pid {}, joined: {:?}) — Ctrl-C to release",
                                args.name, pid, joined
                            );
                            // drained is Vec<serde_json::Value> per Phase-1 D-09 contract.
                            // If --tail, emit each drained envelope as a tail line first, THEN block.
                            if args.tail {
                                for env_value in &drained {
                                    emit_tail_line(env_value, &args.name);
                                }
                                tail_loop(&mut client, &args.name).await?;
                            } else {
                                block_until_disconnect(&mut client).await?;
                            }
                            // If we get here, the broker disconnected cleanly (read returned EOF).
                            delay = std::time::Duration::from_secs(1); // reset on successful run
                        }
                        BusReply::Err { kind: famp_bus::BusErrorKind::NameTaken, .. } => {
                            eprintln!("{} is already registered by another process", args.name);
                            return Err(CliError::NameTaken { name: args.name.clone() });
                        }
                        BusReply::Err { kind, message } => {
                            return Err(CliError::BusError { kind, message });
                        }
                        other => {
                            return Err(CliError::Io { /* unexpected reply */ });
                        }
                    }
                }
                Err(_) => { /* fall through to retry */ }
            }
            // Reconnect or exit
            if args.no_reconnect {
                return Err(CliError::Disconnected);
            }
            eprintln!("broker disconnected — reconnecting in {}s", delay.as_secs());
            tokio::time::sleep(delay).await;
            delay = std::cmp::min(delay * 2, cap);
        }
    }
    ```

    Implement helpers:
    - `async fn block_until_disconnect(client: &mut BusClient) -> Result<(), CliError>` — reads frames from the stream forever, ignoring them (the broker may push delivery events even though default-mode doesn't display them; or, if the broker only replies to messages, this is a `tokio::signal::ctrl_c().await` future that returns on Ctrl-C OR the underlying stream error). **Concrete decision: use `tokio::select!` between `tokio::signal::ctrl_c()` and a forever-pending future; on Ctrl-C, return Ok and let the process exit naturally (Drop closes the UnixStream which the broker treats as Disconnect).**
    - `async fn tail_loop(client: &mut BusClient, identity: &str) -> Result<(), CliError>` — same shape but additionally read inbound frames; on each `BusReply::AwaitOk { envelope }` (or whatever the broker pushes for tail) call `emit_tail_line(&envelope, identity)`. **Note: Phase 1 broker design pushes async deliveries via separate Inbox/Await polling, NOT via unsolicited frames. So `tail_loop` is implemented as a 1-second poll cycle: send `BusMessage::Inbox { since: cursor, include_terminal: false }`, emit each new line, advance local cursor, sleep 1s, repeat. The cursor advance writes to disk via cursor_exec for parity with `famp inbox ack`.**
    - `fn emit_tail_line(envelope: &serde_json::Value, _identity: &str)` — formats per RESEARCH §2 item 5: `< 2026-04-28T14:32:01Z from=<name> to=<name|#chan> task=<uuid> body="<truncated-80-chars>"`. The envelope is canonical JSON with body, `from`, `to`, `task` fields visible at the top level. Body is truncated to 80 chars and quoted with serde_json escaping; ISO-8601 from `time::OffsetDateTime::now_utc().format(...)`.
    - `fn map_bus_client_err(e: BusClientError) -> CliError` — maps BrokerUnreachable / Io to `CliError::BrokerUnreachable` (add this variant to `cli/error.rs` if missing).
