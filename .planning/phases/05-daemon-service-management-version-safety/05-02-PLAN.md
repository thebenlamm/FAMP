<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installer → launchd service file | The daemon WRITES a service file that auto-launches a binary at login. The contents (binary path, env) are the untrusted surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | Path injection — a relative/tilde ProgramArguments path could launch the wrong binary, or a malicious $HOME could redirect | mitigate | generate_plist resolves an ABSOLUTE path via `home.join(".cargo/bin/famp")` (no tilde — launchd does not expand ~); plist_shape test asserts no `~` appears. Guardian gate (Plan 03) re-reviews the literal XML before first load. |
| T-05-05 | Information Disclosure | Service writes secrets to log via EnvironmentVariables | mitigate | Locked shape has NO EnvironmentVariables key; the plist_shape test asserts its absence and guardian enforces it on the literal XML. |
| T-05-06 | Elevation of Privilege | Daemon runs as root | mitigate | User-level LaunchAgent only — no UserName/GroupName key (test asserts absence); installs to the user's own ~/Library/LaunchAgents. |
</threat_model>
