<objective>
Reconcile the human-facing display version to a single honest source of truth
(`0.11.0`) across `famp -V`, the help banner, and the handshake client string
(VER-02), and add client-side enforcement of the bus protocol version handshake
so version skew between a long-lived daemon and an upgraded client fails loud
with an actionable error (VER-01).

Purpose: A long-lived `KeepAlive=true` daemon means a freshly `cargo install`ed
client meets the OLD daemon on every connect until restart. Without a loud,
actionable skew error the user silently misroutes; without a unified version
the skew diagnosis is untrustworthy.

Output: Workspace bumped to 0.11.0, banner unified, `BUS_PROTO_VERSION`
doc-commented (value unchanged), new `BusClientError::ProtocolMismatch` variant
naming `famp daemon restart`, three new unit tests.
</objective>
