<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| status probe → launchctl/inspect | status reads OS registration (exit code only) and broker liveness (connect probe). Read-only; no mutation. |
| restart → launchctl/systemctl | restart kills + relaunches the service from the on-disk binary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Tampering / Information Disclosure | Parsing `launchctl print` free-text could be spoofed or misread (man page: "NOT API") | mitigate | status uses EXIT CODES only (Stdio::null on stdout/stderr); no text parse of launchctl output. Acceptance asserts Stdio::null and absence of a print-output parse. |
| T-05-13 | Tampering | restart picks up whatever binary is on disk at `~/.cargo/bin/famp` | accept | This is the intended DAEMON-05 behavior (pick up a replaced binary). The binary path was locked + guardian-reviewed in the plist (Plan 02/03); restart does not change the path, only relaunches it. |
</threat_model>
