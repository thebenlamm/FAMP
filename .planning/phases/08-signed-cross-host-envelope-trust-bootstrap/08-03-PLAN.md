<objective>
Build the gateway's ingress verification as a pure, transport-agnostic function:
verify_inbound(bytes, &keyring) canonical-decodes inbound bytes, verifies the
Ed25519 signature under the FAMP-sig-v1\0 domain prefix, and requires the sender
principal's pinned key to match — returning a typed SignedEnvelope on success or
one of two loud, distinct RejectReasons on failure. This is WIRE-01 (reject
unsigned/invalid before the local bus) and TRUST-02 (reject unpinned keys) proven
in-process.

Purpose: Phase 9 feeds this function the HTTP request body over the live
transport; Phase 8 delivers and unit-tests the function itself (data-as-input,
D-07). It builds no socket and touches no bus.
Output: verify.rs (verify_inbound), RejectReason enum, keyring/envelope deps,
four unit tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inbound bytes → gateway | Untrusted cross-host envelope bytes enter verify_inbound; this is the protocol-grade boundary where INV-10 is enforced before the local bus. |
| keyring file → verify | Pinned peer keys are the trust root; only keys placed there by explicit import are trusted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-06 | Spoofing | Envelope signature stripping / forgery | critical | mitigate | verify_inbound requires SignedEnvelope::decode (verify_strict) against the pinned key; unsigned/tampered ⇒ InvalidSignature, no bus write (rejects_unsigned/rejects_bad_signature). |
| T-08-07 | Spoofing | Unpinned-key implicit trust (TOFU bypass) | critical | mitigate | keyring.get(None) ⇒ hard UnpinnedKey reject; no auto-pin at receive time (rejects_unpinned_key). |
| T-08-08 | Tampering | Duplicate object-key JSON smuggling | high | mitigate | peek_sender + SignedEnvelope::decode both route through from_slice_strict (duplicate-key rejection at any depth) — inherited from the canonical substrate. |
| T-08-09 | Repudiation | Collapsed reject reasons hide tamper-vs-unknown-peer | medium | mitigate | Two distinct RejectReason variants (D-08); operator/Phase-9 4xx can distinguish tampering from missing import. |
| T-08-SC | Tampering | Dependency supply chain | low | accept | famp-keyring + famp-envelope are internal workspace members already in Cargo.lock; no new external package (RESEARCH Package Legitimacy Audit). |
