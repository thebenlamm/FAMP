<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator clipboard/Signal → import | The out-of-band human transport of the export blob; paste corruption or a wrong paste is the risk (fingerprint eyeball-check mitigates). |
| import → peer keyring file | Pinning writes the trust root read by verify_inbound; a wrong/conflicting pin must fail closed. |
| gateway key file on disk | The gateway's own signing identity; persisted once, must not be silently regenerated (which would break already-distributed trust). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-10 | Spoofing | Wrong/tampered key pasted during import | high | mitigate | key_id fingerprint field re-derived on import and compared; human reads last chars aloud over Signal (CONTEXT <specifics>). |
| T-08-11 | Tampering | Silent trust overwrite on conflicting re-pin | high | mitigate | pin_tofu fails closed on a different key for a pinned principal; mapped to a loud key-conflict CliError (round-trip test asserts). |
| T-08-12 | Spoofing | Gateway key silently regenerated, invalidating distributed trust | medium | mitigate | load_or_generate reads the existing file when present; generate-once, then load (idempotency unit test). |
| T-08-13 | Information Disclosure | Signing key material leaked over FAMP | high | mitigate | Only the PUBLIC key + fingerprint are exported; import reads local file/stdin; no private key ever printed or sent. |
| T-08-SC | Tampering | Dependency supply chain | low | accept | famp-keyring is an internal workspace member already in Cargo.lock (promotion dev→real dep only); no new external package (RESEARCH Package Legitimacy Audit). |
</threat_model>
