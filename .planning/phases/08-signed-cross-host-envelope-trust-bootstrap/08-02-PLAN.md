<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSPRNG ↔ signing key | Key entropy source; a weak source would produce guessable keys. |
| pubkey bytes ↔ key_id string | Fingerprint derivation; a human compares key_id out-of-band to detect paste corruption. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-04 | Spoofing | Signing key generated from weak/non-CSPRNG entropy | high | mitigate | generate() uses OsRng only (Don't Hand-Roll row); no fixed-seed path in production code. |
| T-08-05 | Repudiation | key_id collision treated as key identity | low | mitigate | key_id documented as non-anchor; trust decision always uses the full pinned pubkey, never key_id. |
| T-08-SC | Tampering | rand dependency supply chain | low | accept | rand is already vendored workspace-wide and in Cargo.lock; adding it to one more member introduces no new supply-chain surface (RESEARCH Package Legitimacy Audit — no new external packages). |
</threat_model>
