## Provenance of §7.1c bytes (PITFALLS P10)

Both the canonical JSON and the Ed25519 signature were computed from an
external reference toolchain — **zero code in this repo touched the bytes**.

- **RFC 8785 canonicalization tool:** Python `jcs` package, version `0.2.1`
  (<https://pypi.org/project/jcs/>).
- **Ed25519 signing tool:** Python `cryptography` package, version `46.0.7`
  (`cryptography.hazmat.primitives.asymmetric.ed25519.Ed25519PrivateKey`).
- **Keypair source:** RFC 8032 §7.1 Test 1, secret key
  `9d61b19deffd5a60ba844af492ec2cc44449c5697b326919703bac031cae7f60`.
- **Public key cross-check:** derived `d75a980182b10ab7d54bfed3c964073a0ee172f3daa62325af021a68f707511a` matches RFC 8032 §7.1 Test 1 byte-for-byte.
- **Command used (tested 2026-04-12):**
  ```sh
  python3 -m venv /tmp/famp-vec
  /tmp/famp-vec/bin/pip install jcs==0.2.1 cryptography==46.0.7
  /tmp/famp-vec/bin/python compute-vector-01.py   # scratch script
  ```
  The full inline script is recorded in §7.1c.0 of the spec so any reviewer
  can reproduce independently. The scratch venv and script were deleted
  after use — per the plan's instruction not to commit them.
- **Final outputs:**
