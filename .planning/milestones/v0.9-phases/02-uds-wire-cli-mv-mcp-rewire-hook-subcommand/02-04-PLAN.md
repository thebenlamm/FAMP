<objective>
Rewire `famp send` (CLI-02) from v0.8's HTTPS-via-`famp listen` transport to v0.9's UDS-via-`famp broker` transport. Preserve every existing flag and behavior — only the transport layer swaps. Identity binding follows D-10: connection-level via `Hello { bind_as: Some(resolved_identity) }`, NOT a per-message field. The `BusMessage::Send` wire shape is unchanged from Phase 1.

This plan does NOT modify `crates/famp-bus/src/proto.rs` (the original plan added `act_as: Option<String>`; D-10 rejects that approach in favor of `Hello.bind_as` for ALL one-shot CLI commands uniformly). The proto.rs `Hello.bind_as` field was added in plan 02-02.

Purpose: This is the per-message user surface. Plan 02-03's `famp register` holds the identity slot; plan 02-04's `famp send` rides on it via Hello.bind_as proxy. The MCP `famp_send` tool (plan 02-09) calls into `run_at_structured` so changes here propagate to MCP automatically.

Output: Modified `crates/famp/src/cli/send/{mod,client}.rs` (transport swap + `--as` flag wired into Hello.bind_as on connect).
</objective>

       ```rust
       pub async fn run_at_structured(sock: &Path, args: SendArgs) -> Result<SendOutcome, CliError> {
           // 1. Resolve identity (D-01) for the Hello.bind_as proxy
           let identity = crate::cli::identity::resolve_identity(args.act_as.as_deref())?;
           // 2. Build target
           let target = match (args.to.as_deref(), args.channel.as_deref()) {
               (Some(name), None) => Target::Agent { name: name.to_string() },
               (None, Some(ch)) => Target::Channel { name: normalize_channel(ch)? },
               (Some(_), Some(_)) => return Err(CliError::SendArgsInvalid { reason: "--to and --channel are mutually exclusive".into() }),
               (None, None) => return Err(CliError::SendArgsInvalid { reason: "exactly one of --to or --channel is required".into() }),
           };
           // 3. Build the envelope using the existing fsm_glue helpers (preserves --new-task/--task/--terminal matrix verbatim)
           let envelope = crate::cli::send::fsm_glue::build_envelope_value(&args)?;
           // 4. Connect bus with Hello.bind_as = Some(identity) per D-10 — this is the
           //    proxy binding mechanism. The broker validates at Hello time that
           //    `identity` is held by a live `famp register` process; if not, returns
           //    HelloErr{NotRegistered}, which BusClient::connect surfaces as
           //    BusClientError::HelloFailed{NotRegistered, ..}.
           let mut bus = crate::bus_client::BusClient::connect(sock, Some(identity.clone())).await
               .map_err(|e| match e {
                   crate::bus_client::BusClientError::HelloFailed { kind: famp_bus::BusErrorKind::NotRegistered, .. } => {
                       CliError::NotRegisteredHint { name: identity.clone() }
                   }
                   _ => CliError::BrokerUnreachable,
               })?;
           // 5. Send the message — NO `act_as` field; broker stamps `from` based on
           //    effective_identity(state) which resolves to the bound proxy name (D-10).
           let reply = bus.send_recv(BusMessage::Send {
               to: target,
               envelope,
           }).await.map_err(|e| CliError::BusClient { source: format!("{e:?}") })?;
           match reply {
               BusReply::SendOk { task_id, delivered } => Ok(SendOutcome {
                   task_id: task_id.to_string(),
                   delivered: format!("{delivered:?}"),
               }),
               BusReply::Err { kind: BusErrorKind::NotRegistered, .. } => {
                   // Per-op liveness re-check failed (the holder died between Hello and Send)
                   Err(CliError::NotRegisteredHint { name: identity })
               }
               BusReply::Err { kind, message } => Err(CliError::BusError { kind, message }),
               other => Err(CliError::Io { /* unexpected reply */ }),
           }
       }
       ```
       (The wildcard `other =>` arm is on `BusReply` variants we don't expect for a Send op — `BusErrorKind` is exhaustively matched in the prior arms; this is the v0.8 pattern.)
