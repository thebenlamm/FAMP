  <behavior>
    Builds the reusable helpers the five integration tests consume. Keeping them in
    `tests/common/mod.rs` follows Rust's convention for cross-test-binary shared code (Cargo won't
    treat `common/` as its own test binary because it's a subdirectory, not a top-level file).

    Helpers to ship:

    1. `pub fn spawn_listen(home: &Path, listen_addr: SocketAddr) -> std::process::Child`
       - Runs `Command::new(env!("CARGO_BIN_EXE_famp"))` with args `["listen", "--listen",
         &listen_addr.to_string()]` and env `FAMP_HOME=<home>`.
       - Captures stderr via `Stdio::piped()` so tests can assert on "listening on https://..." line.
       - Returns the `Child`.
       - Does NOT wait for the daemon to bind — the caller polls (next helper).

    2. `pub fn wait_for_bind(child: &mut Child, addr: SocketAddr, timeout: Duration) -> io::Result<()>`
       - Polls `TcpStream::connect_timeout(&addr, ...)` in a loop with 50ms sleeps until success or
         timeout. Returns `Err` if timeout expires. Used as a synchronization point so tests can send
         requests without racing the bind.

    3. `pub fn init_home(home: &Path)`
       - Spawns `famp init` as a subprocess against `FAMP_HOME=home` and waits for exit. Used as the
         setup step for every test.

    4. `pub async fn post_signed_envelope(addr: SocketAddr, home: &Path, body_str: &str)
       -> reqwest::Result<reqwest::Response>`
       - Reads `home.join("key.ed25519")` as the signing seed, builds a `FampSigningKey`,
         constructs a minimal `SignedEnvelope<AckBody>` (self → self, scope=Standalone,
         class=Ack, Advisory authority), signs via `.sign(&sk)`, encodes via `.encode()`, and POSTs
         the bytes to `https://{addr}/famp/v0.5.1/inbox/{principal}` where `principal` is the
         base64url-unpadded pubkey derived from the same seed.
       - Uses a `reqwest::ClientBuilder` that loads the daemon's `tls.cert.pem` as a trust anchor
         via `reqwest::Certificate::from_pem` — same shape as the v0.7 `cross_machine_two_agents`
         example.
       - Returns the raw `reqwest::Response` so tests can assert on status.

    5. `pub fn read_inbox_lines(home: &Path) -> Vec<serde_json::Value>`
       - Thin wrapper around `famp_inbox::read::read_all(home.join("inbox.jsonl")).unwrap()`.

    File header: `#![allow(dead_code, clippy::unwrap_used, clippy::expect_used, unused_crate_dependencies)]`
    because different test binaries use different subsets of these helpers (dead_code silences the
    unused-per-binary warnings).
  </behavior>

    **`listen_durability.rs`** — one test named `sigkill_after_200_leaves_line_intact`:
    - MUST use a real subprocess (SIGKILL cannot be sent to an in-process tokio task in a
      meaningful way). `spawn_listen` with an ephemeral port — but ephemeral ports require reading
      the bound addr from the child's stderr. The daemon prints `listening on https://127.0.0.1:<port>`
      — the test reads the child's stderr line by line until it parses that line, extracts the port,
      and proceeds.
    - POST a signed envelope via `post_signed_envelope` (async).
    - Immediately on receiving the 200: `child.kill().unwrap()` (sends SIGKILL on unix).
    - Wait for the child to exit, read `home.join("inbox.jsonl")` via `read_all`, assert one value
      present. This proves `sync_data` happened before the 200.
    - NOTE: the test uses `--listen 127.0.0.1:<port>` where port is a random free port obtained via
      `std::net::TcpListener::bind("127.0.0.1:0"); listener.local_addr()?.port(); drop(listener);`.
      This is racy (another process could grab the port in the window) but acceptable for a local
      test harness. Better approach: have `famp listen` print the bound port to stderr — which it
      does — and read it back from the child's stderr pipe. Use the stderr-pipe approach for
      durability (no race window); use the "bind+drop" approach only as a fallback if stderr parsing
      fails.

    **`listen_bind_collision.rs`** — one test named `second_listen_on_same_port_errors_port_in_use`:
    - Subprocess A: spawn_listen on a chosen port, wait_for_bind.
    - Subprocess B: spawn_listen on THE SAME port.
    - Wait up to 5s for B to exit; assert exit code is non-zero AND stderr contains the string
      "already bound" (the error message in `CliError::PortInUse` per Task 1 of Plan 02-02).
    - Kill subprocess A; both cleaned up.

    **`listen_shutdown.rs`** — one test named `sigint_causes_exit_0_within_5s`:

    - Send SIGINT via `nix::sys::signal::kill(Pid::from_raw(child.id() as i32), Signal::SIGINT)` —

      only. Or even simpler: invoke `/bin/kill -INT <pid>` via `Command`. Use the `/bin/kill` shell
      approach to keep the test source free of `unsafe`.

    - On timeout, SIGKILL the child and fail the test.

  <action>
    Create the five test files per the `<behavior>` spec. Execution order within each file is:
    init_home → spawn or in-process run_on_listener → exercise → assert → cleanup.

    Pragmatic notes the executor must handle:

    1. **Ephemeral port read-back via stderr.** The daemon's stderr format is fixed at
       `listening on https://127.0.0.1:<port>\n` — parse with a regex or simple string split on
       `:` from the right. If `BufReader::read_line` on the child's stderr pipe hangs, the child has
       deadlocked and the test should time out.

    2. **Test cleanup.** Every spawned child MUST be killed on test exit (even on panic) to avoid
       leaked daemons. Use a `Drop` guard struct around the `Child`:
       ```rust
       struct ChildGuard(Option<Child>);
       impl Drop for ChildGuard {
           fn drop(&mut self) {
               if let Some(mut c) = self.0.take() { let _ = c.kill(); let _ = c.wait(); }
           }
       }
       ```

    3. **Parallel isolation.** Each test gets its own `TempDir` and therefore its own FAMP_HOME,
       its own inbox, and its own identity. Port collisions are avoided by using ephemeral ports
       everywhere except `listen_bind_collision.rs`, which specifically wants two daemons on the
       same port — that test picks a fixed port by binding+dropping a scratch `TcpListener` to grab
       a free port and immediately reusing it. Accept the race window for the bind-collision test.

    4. **reqwest self-signed cert.** The daemon ships its own `tls.cert.pem`; the test client reads
       that file from `FAMP_HOME` and feeds it to `reqwest::Certificate::from_pem`, then
       `ClientBuilder::add_root_certificate(cert).danger_accept_invalid_hostnames(true)` — because
       the cert has SAN `localhost` but the test connects to `127.0.0.1`. Match the pattern in the
       v0.7 `cross_machine_two_agents` example.

    5. **SIGINT delivery via /bin/kill.** Use
       `std::process::Command::new("kill").args(["-INT",
       &child.id().to_string()]).status()?` — cross-platform enough for unix, keeps the test source
       safe-rust. Mark the test `#[cfg(unix)]` and skip on non-unix (v0.8 is unix-only per CONTEXT).

    6. **The durability test is the most fragile.** Recommended sequence:
       - spawn_listen with stderr piped
       - read stderr until "listening on https://" line parsed
       - post_signed_envelope (async, via `#[tokio::test]`)
       - IMMEDIATELY after the `await` on the response returning 200: `child.kill()?`
       - `child.wait()` for the child to actually exit (kill is async at the OS level)
       - Only THEN open the inbox file and call `read_all`
       - Assert exactly 1 line and it parses.

    7. **Wait, run_on_listener vs subprocess.** The smoke test can use in-process
       `run_on_listener` for speed. But durability, bind-collision, and shutdown tests MUST use a
       subprocess because those contracts are only meaningful at the OS-process level. Keep the
       distinction explicit in each file's module doc.
  </action>
