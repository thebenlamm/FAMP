<objective>
Ship `famp mcp` — a stdio JSON-RPC MCP server exposing four tools that wrap
Phase 3's CLI entry points, with structured `famp_error_kind` discriminators
on every failure.

Purpose: Claude Code can only drive `famp` through MCP tools (no shell
access). The four tools are the contract the two manual-smoke sessions speak.
Structured errors let Claude Code react to `peer_not_found` differently from
`await_timeout`.

Output:
- `cli::mcp` module tree + clap subcommand `Mcp` + dispatcher wiring
- Four tool handlers that thin-wrap existing `cli::{send, await_cmd, inbox, peer}` code
- `CliError::mcp_error_kind()` exhaustive-match method
- Two integration tests: the exhaustive kind test and a stdio subprocess test
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code → famp mcp stdio | JSON-RPC requests crossing a trusted local pipe; input shapes validated by tool schemas. |
| famp mcp → cli::* entry points | Internal boundary; already-typed Rust calls. |
| famp mcp error path → JSON-RPC error response | Typed `CliError` serialized via `mcp_error_kind()` — no raw `Display` leakage of sensitive state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Tampering | JSON-RPC input schemas | mitigate | Every tool input deserializes via `serde` with `deny_unknown_fields`; malformed inputs return `invalid_params`. No `serde_json::Value` pass-through to downstream CLI functions. |
| T-04-11 | Information Disclosure | Error messages | mitigate | `Display` impls on `CliError` already redact key material (IDENT-06 gate from Phase 1). `mcp_error_kind()` returns a discriminator, not a message — structured consumers can react without seeing the Display output. `details: {}` is an empty map in v0.8 to prevent accidental leakage; populated explicitly per-variant in a future plan if needed. |
| T-04-12 | Repudiation | tool call audit | accept | v0.8 has no MCP audit log. Claude Code's own tool-call log is the evidence trail for the manual smoke. Federation Profile adds server-side audit. |
| T-04-13 | Elevation of Privilege | compile-time error_kind gate | mitigate | Non-exhaustive match on `CliError` in `error_kind.rs` is a hard compile error under the workspace's `deny(unreachable_patterns)` / default rustc settings. Adding a variant without a mapping cannot ship. |
| T-04-14 | Denial of Service | stdio pipe flood | accept | Local single-user process, no network exposure. Out of scope. |
</threat_model>
