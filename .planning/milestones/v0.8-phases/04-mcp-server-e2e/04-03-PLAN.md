<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| two in-process daemons, same machine | Each daemon has its own identity + peers.toml; they communicate over localhost HTTPS like cross-machine peers would. |
| manual smoke pipes | Claude Code stdio → `famp mcp` → `cli::*` → inbox files. All on the same laptop, localhost only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-20 | Spoofing | two_daemon_harness principals | mitigate | Each daemon uses a distinct principal (`agent:localhost/alice`, `agent:localhost/bob`). Their keyrings contain only the other principal + self. An envelope claiming to be from an unrelated principal is rejected by the multi-entry keyring check (locked by Plan 04-01 Task 1's unknown-principal test). |
| T-04-21 | Tampering | mutual TOFU bootstrap | accept | First-contact TLS fingerprint capture is trust-on-first-use. An attacker who MITMs the very first handshake can pin their own fingerprint. On localhost this is not a realistic threat; federation deployment (v0.9) will replace TOFU with Agent Cards. |
| T-04-22 | Denial of Service | two-daemon test port allocation | accept | Tests use ephemeral 127.0.0.1:0 ports. Parallel nextest execution could in principle race on port reuse, but `std::net::TcpListener::bind(127.0.0.1:0)` yields unique ports via the kernel. |
| T-04-23 | Repudiation | manual smoke checklist | mitigate | Checklist captures operator identity, date, outcome, qualitative notes, and archives inbox.jsonl artifacts under `smoke-artifacts/`. The raw envelope bytes in inbox.jsonl are signature-verified — the archive is non-repudiable evidence of the run. |
</threat_model>
